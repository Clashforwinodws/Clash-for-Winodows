# 贡献指南

感谢你对 Clash 下载站的关注！我们欢迎所有形式的贡献，包括但不限于文档改进、Bug 反馈、代码提交以及使用经验分享。

## 如何报告问题
- 请先在 [Issues](https://github.com/Clashforwinodws/Clash-for-Winodows/issues) 中搜索是否已有相同问题。
- 使用清晰的标题和描述，附上复现步骤、环境信息（操作系统、客户端版本）以及截图或日志。

## 如何提交改进
1. Fork 本仓库。
2. 创建你的特性分支 (`git checkout -b feature/amazing-improvement`)。
3. 提交变更 (`git commit -m '添加某个改进'`)。
4. 推送到分支 (`git push origin feature/amazing-improvement`)。
5. 打开一个 Pull Request，并详细说明改动内容和原因。

## 文档与教程贡献
- 文档使用 Markdown 编写，请保持格式一致。
- 教程页面请确保步骤清晰、无遗漏，并且与实际客户端行为相符。
- 欢迎翻译、补充 FAQ 或更新过时内容。

## 行为准则
- 保持友善和尊重，禁止人身攻击或无关言论。
- 我们遵循 [Contributor Covenant](https://github.com/Clashforwinodws/Clash-for-Winodows/blob/main/SECURITY.md) 行为准则。

## 风格指南
- 中英文混排时，建议在英文单词前后添加空格。
- 代码块使用正确的语法高亮标识。
- 提交信息使用中文或英文均可，但请保持简洁明了。

再次感谢你的贡献！如果你对项目有任何疑问，可以随时在 Discussions 或 Issues 中发起讨论。
